Acceptance criteria

### AC-1: Verify Equipment Identification Tab UI and Field Presence

**Given** the user is on the Equipment List page  
**When** the user clicks on "+ New Equipment" button  
**Then** the system should open the Add Equipment form  
**And** the "Equipment Identification" tab should be selected by default  

**And** the following fields should be visible:

- TPS ID (Auto-generated, Read-only)
- Status (Default: ACTIVE)
- Equipment Category (Dropdown)
- Equipment Type (Dropdown)
- Manufacturer (*) (Searchable / Add New)
- Supplier / Vendor (*) (Searchable / Add New)
- Supplier Identification Number (*) (Text Field)
- Feature (Optional)
- Model Information (Optional)

**And** all fields should be properly aligned and accessible  
**And** mandatory fields should be marked with "*"  
**and** mandatory feidls should show the validation if user try to save without adding the data 

### AC-2: Verify TPS ID Auto Generation

**Given** the user opens the Add Equipment form  
**Then** the TPS ID should be auto-generated by the system  

**And** the TPS ID field should:
- Be unique
- Be non-editable (read-only)
- Follow the system-defined format (e.g., E11037)

### AC-3: Validate Mandatory Fields in Equipment Identification Tab

**Given** the user is on the Equipment Identification tab  
**When** the user clicks "Save" without filling mandatory fields  
**Then** the system should display validation messages:

- "Manufacturer is required"
- "Supplier is required"
- "Supplier Identification Number is required"

**And** the corresponding fields should:
- Be highlighted with red border
- Show inline validation error messages

**And** the record should NOT be saved

### AC-4: Verify Manufacturer Field Functionality

**Given** the user interacts with the Manufacturer field  
**Then** the field should allow:
- Searching existing manufacturers
- Selecting from dropdown
- Adding a new manufacturer (free text)

**And** the field should be mandatory  
**And** validation should trigger if left empty

### AC-5: Verify Supplier / Vendor Field Functionality

**Given** the user interacts with the Supplier / Vendor field  
**Then** the field should allow:
- Searching existing suppliers
- Selecting from dropdown
- Adding a new supplier

**And** the field should be mandatory  
**And** validation should trigger if left empty

### AC-6: Validate Supplier Identification Number Field

**Given** the user enters Supplier Identification Number  
**Then** the system should:
- Accept valid input (e.g., V-1001-A)
- Ensure uniqueness across records

**When** the field is empty and user clicks Save  
**Then** validation message should appear:
- "Supplier Identification Number is required"

**And** once created:
- The field should NOT be editable

### AC-7: Validate Optional Fields

**Given** the user is filling the form  
**Then** the following fields should be optional:
- Feature
- Model Information

**And** the system should allow saving without these fields

### AC-8: Verify Default Status

**Given** the user opens Add Equipment form  
**Then** the Status field should be defaulted to "ACTIVE"  

**And** the user should be able to change it if applicable


Business Rule : 
## Business Rules: Equipment Identification

BR-1: System shall auto-generate a unique TPS ID for every new equipment record.  
BR-2: TPS ID shall be read-only and non-editable by the user.  

BR-3: Manufacturer, Supplier / Vendor, and Supplier Identification Number are mandatory fields.  

BR-4: System shall not allow saving the record if any mandatory field is empty.  

BR-5: System shall display validation messages and highlight mandatory fields when validation fails.  

BR-6: Manufacturer and Supplier / Vendor fields shall support:
- Search existing records
- Selection from master data
- Creation of new entries  

BR-7: Supplier Identification Number shall be unique for each equipment record.  

BR-8: Supplier Identification Number shall become non-editable after record creation.  

BR-9: Status field shall default to "ACTIVE" on new record creation.  

BR-10: Equipment Category and Equipment Type shall be selectable from predefined dropdown values.  

BR-11: Feature and Model Information fields shall be optional.  

BR-12: System shall allow saving the record when all mandatory fields are valid, regardless of optional field values.
